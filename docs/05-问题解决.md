# 1、误刷 ROS2 导致软件端无法连接
### 背景
我想要将其配成ubuntu22.04+ROS2 ,但是直接把官网上的ROS2更新文件通过软件设置-version扔进去了，导致出现冲突。软件端无法点击连接，终端还可以。
### 补充
Ubuntu20.04+ROS noteic
ubuntu22.04+ROS humble
要修改的话建议获取对应镜像后进行刷机，技术人员说要寄回去他们弄，于是放弃。

### 恢复原先配置具体步骤
![[580d2f924aa08a605b1354aded8c5b9c.jpg]]
1. 到官网下载ROS更新文件，解压
2. 直接打开终端（注意不用SSH），输入：
   ```
scp D:\viobot2-resource\ros1\baton_ros_update_251204\update.tar root@192.168.1.100:/home/user/update/
   ```
具体说明：先输入scp， 按空格，将刚刚解压好的tar文件（不是zip)拖到终端，可以自动复制地址，然后再按空格，复制后面那一串
3. 会要求你输入密码![[Pasted image 20260323172616.png]]
4. 进入SSH
   ```
# 确认一下文件到没到
ls -lh /home/user/update/

# 执行修复脚本
/etc/other_setup/update.sh
   ```

# 一、建图与重定位

1. 设置中点击菜单栏中的“loop”，勾选”记录建图信息“，点击”确定“
2. 点击“保存BOW”，在点击“save”
3. 点击“开始建图”，等待完成100%，再点击“关闭”
4.  设置中点击菜单栏中的“loop”，去掉”记录建图信息“，勾选”开启回环/重定位“，和”加载地图“，点击”确定“
5. 手动启动算法并加载地图，终端你可以查看话题数据
```

rostopic echo /baton/stereo3/odom_relo
```

1. 出现的乱码猜测是应为GND接线掉了，导致
	- 信号电平不稳定
	- 数据帧错位
	- 出现乱码或 `^@`（ASCII 0 字符）
2. 关于日志输出的解释
   ```
   opt_flow_ptr->input_queue -1 
opt_flow_ptr->output_queue 0 
out_state_queue -1 
imu_data_queue -1
   ```

	这是 stereo3 节点内部的**队列状态信息**，用于调试和数据流监控。
	 各队列含义

| 队列名称                         | 作用       | 数值含义                |
| ---------------------------- | -------- | ------------------- |
| `opt_flow_ptr->input_queue`  | 光流算法输入队列 | `-1` 表示队列为空或未初始化    |
| `opt_flow_ptr->output_queue` | 光流算法输出队列 | `0` 表示队列中有 0 个待处理结果 |
| `out_state_queue`            | 状态输出队列   | `-1` 表示队列为空         |
| `imu_data_queue`             | IMU 数据队列 | `-1` 表示队列为空         |
 数值含义
- **`-1`**：队列为空或未初始化（正常，等待数据）
- **`0`**：队列中有 0 个数据（正常）
- **正数**：队列中有等待处理的数据（临时）
- **很大正数**：队列积压，可能算力不足
#### 关于队列日志的优化（可选）

如果觉得这些日志干扰，可以调整日志级别：
```bash
# 运行时只显示 INFO 以上级别
rosrun baton stereo3_node __log_level:=INFO
```

或在脚本中修改启动方式：
```bash
# 屏蔽 DEBUG 日志
/root/Baton/install/lib/stereo3/stereo3 2>&1 | grep -v "queue"
```
